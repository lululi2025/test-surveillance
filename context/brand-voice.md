## 1. 品牌語氣定義

### 一句話描述

> **Professional, confident, approachable** — 用簡潔語言解釋複雜技術，讓讀者覺得「這家公司懂我的問題，而且解法不複雜」。

### 語氣光譜

```
技術冷硬 ◻◻◼◼◻ 親切口語
          ↑ EnGenius 落點
```

EnGenius 的語氣在光譜上偏向專業端，但不到學術或規格書的冷硬程度。目標是讓 IT 管理者覺得「這品牌有技術底子」，同時讓非技術決策者也能讀懂。

## 2. 品牌語氣四大特質

### 2.1 Confident, not aggressive（自信但不咄咄逼人）

直接陳述產品能力，不需要貶低競品或使用誇大語言。

| ✅ Do | ❌ Don't |
|-------|---------|
| "Enterprise-grade networking without mandatory licensing fees." | "Unlike our competitors who rip you off with licensing..." |
| "Deploy new sites in minutes with zero-touch provisioning." | "The fastest, most revolutionary deployment ever!!!" |

### 2.2 Clear, not oversimplified（清晰但不過度簡化）

技術名詞可以用，但要確保讀者能從上下文理解含義。不需要迴避專業用語，但也不堆砌規格。

| ✅ Do | ❌ Don't |
|-------|---------|
| "Built-in NPU processes video at the edge — no bandwidth strain, just smart security." | "Uses neural processing unit with 6 TOPS INT8 inference throughput on H.265 codec..." |
| "Cloud-managed access points with zero-touch provisioning" | "Wi-Fi boxes you control from the internet" |

### 2.3 Helpful, not salesy（有幫助而非推銷感）

以解決問題為導向，不是以推銷產品為導向。先講痛點，再講解法。

| ✅ Do | ❌ Don't |
|-------|---------|
| "Managing multi-site networks shouldn't require truck rolls. Deploy and monitor remotely from one dashboard." | "Buy our amazing cloud platform today! Limited time offer!" |
| "Convert bulky video into streamlined metadata, enabling instant object detection." | "Our incredible AI cameras are the best on the market!" |

### 2.4 Grounded, not hype-driven（務實而非炒作）

用數據和具體場景支撐論點，避免空泛的形容詞。AI 相關描述尤其需要克制。

| ✅ Do | ❌ Don't |
|-------|---------|
| "AI-powered object detection at the edge — reducing bandwidth by processing video on-device." | "Game-changing AI that will transform everything!" |
| "Allied Market Research predicts API market growing at a CAGR of 20.3%." | "Everyone knows the future is API-driven." |

## 3. 寫作結構模式

### 官網觀察到的一致模式：Problem → Solution → Benefit

EnGenius 的文案幾乎都遵循這個結構：

```
[痛點/現狀] → [EnGenius 怎麼解決] → [帶來什麼好處]
```

**範例拆解：**

| 步驟 | 範例 |
|------|------|
| Problem | 傳統網路管理需要到場部署，耗時耗力 |
| Solution | EnGenius Cloud 提供 zero-touch provisioning |
| Benefit | 從任何地方遠端部署新據點，幾分鐘內上線 |

**組合成文案：**
> "Stop sending technicians to every new site. With zero-touch provisioning, deploy and configure remotely — new locations go live in minutes, not days."

## 4. 英文文案規範

### 句型偏好

| 原則 | 說明 | 範例 |
|------|------|------|
| 主動語態為主 | 主詞做動作，不是被動接受 | "EnGenius Cloud **manages** your network" ✅ / "Your network **is managed** by..." ❌ |
| 短句開頭 | 用短句抓注意力，再用中句補充 | "No mandatory licensing. Deploy enterprise-grade Wi-Fi with the freedom to scale — without annual fees eating into your budget." |
| 動詞驅動 | 用強動詞代替 be 動詞 + 形容詞 | "**Deploy** in minutes" ✅ / "It is quick to deploy" ❌ |
| 具體 > 抽象 | 用數字或場景取代模糊描述 | "Manage 500 sites from one dashboard" ✅ / "Manage many sites easily" ❌ |

### 常用品牌動詞（Verb Bank）

從官網文案中提取的高頻動詞，寫文案時優先選用：

```
deploy / manage / monitor / configure / scale
enable / empower / streamline / simplify / unify
deliver / provide / integrate / extend / customize
convert / detect / process / protect / connect
```

### 常用品牌形容詞（Adjective Bank）

```
cloud-managed / AI-powered / enterprise-grade / unified / centralized
real-time / secure / smart / scalable / reliable
simple / affordable / flexible / seamless / robust
```

### 避免使用的詞彙

| 避免 | 原因 | 替代 |
|------|------|------|
| revolutionary / game-changing | 過度炒作 | effective / powerful |
| best-in-class（行銷文案中） | 太泛濫，缺乏差異化 | 用具體數據或功能描述 |
| cutting-edge / next-gen | 空泛 | 描述具體的技術能力 |
| cheap / budget | 暗示品質低 | cost-effective / affordable |
| easy（單獨使用） | 太模糊 | "set up in minutes" / "deploy without on-site visits" |
| synergy / leverage（名詞） | 企業空話 | 用具體動作描述 |

## 5. 中文文案規範

### 基本原則

| 原則 | 說明 |
|------|------|
| 保留英文技術名詞 | Wi-Fi、AP、PoE、SD-WAN、API 等不翻譯 |
| 產品名稱不翻譯 | EnGenius Cloud、FitXpress、DuraFon 維持英文 |
| 避免直譯英文結構 | 不要寫「被設計來...」，改用「專為...打造」 |
| 口語化但不隨便 | 可以用「不用再...」但不用「超讚的」 |

### 中文語氣範例

| 場景 | ✅ Do | ❌ Don't |
|------|-------|---------|
| 功能介紹 | 「透過 EnGenius Cloud 單一平台，集中管理所有據點的網路設備。」 | 「EnGenius Cloud 是一個被設計來讓使用者可以管理網路的平台。」 |
| 痛點描述 | 「還在為每個新據點派人到場安裝？」 | 「傳統的網路部署方式存在諸多不便之處。」 |
| 價值主張 | 「企業級效能，不綁年約授權費。」 | 「我們的產品非常棒而且不用付授權費喔！」 |
| AI 描述 | 「內建 AI 晶片，在裝置端即時辨識物件，不佔頻寬。」 | 「革命性的 AI 技術將徹底改變監控產業！」 |

### 中英混用規則

```
✅ 「支援 zero-touch provisioning，遠端部署不需到場。」
✅ 「透過 API 與現有的 RMM 工具整合。」
❌ 「支援零接觸佈建（zero-touch provisioning）功能，可以遠端進行部署作業。」（過度冗長）
❌ 「用 ZTP 配 RMM 做 integration。」（縮寫過多，失去可讀性）
```

## 6. 不同素材的語氣調整

品牌語氣是一致的，但「刻度」隨素材類型微調：

| 素材類型 | 技術深度 | 語氣調整 | 範例風格 |
|----------|---------|----------|----------|
| Landing Page | 中 | 簡潔有力，benefit-led | "Cloud-managed networking. No licensing fees. Deploy in minutes." |
| Product Page | 中高 | 功能明確，搭配規格 | "ECW336 — Wi-Fi 7 Indoor AP, 4x4 MU-MIMO, Cloud-managed with zero-touch provisioning." |
| Blog / Article | 中 | 教育性，Problem-Solution 結構 | "What is an API? Think of it as a universal translator between applications..." |
| EDM | 低～中 | 短句、強 CTA | "Your network, managed from anywhere. See how →" |
| Social Media | 低 | 最簡短、視覺搭配 | "500 sites. One dashboard. Zero licensing fees." |
| White Paper | 高 | 技術嚴謹、數據佐證 | "The EnGenius Cloud API leverages RESTful architecture..." |
| Brochure | 中 | 圖文平衡、掃讀友善 | Headline + 3 bullet points + spec table |
| Exhibition Backwall | 低 | 視覺優先、slogan 級短句 | "Cloud-Managed. AI-Powered. License-Free." |
| Case Study | 中 | 故事性、數據結尾 | "Before EnGenius: 3 hours per site visit. After: remote deployment in 15 minutes." |
| Partner Materials | 低～中 | 商業導向、利潤強調 | "Better margins. Happier clients. Zero licensing fees to pass through." |

## 7. Headline 公式

經常可套用的標題結構：

```
1. [Benefit] + without + [Pain Point]
   → "Enterprise networking without mandatory licensing."

2. [Action verb] + [Object] + from [Advantage]
   → "Manage every site from a single cloud dashboard."

3. [Number] + [Object]. [Short benefit].
   → "500 sites. One platform. Zero licensing fees."

4. Stop + [Pain Point]. Start + [Benefit].
   → "Stop paying annual licenses. Start scaling freely."

5. [Product] + that + [Benefit verb]
   → "AI cameras that detect, not just record."
```

## 8. 品牌語氣 Checklist

寫完文案後，用以下清單自我檢查：

- [ ] **讀起來像在幫忙解決問題，還是在推銷產品？** → 應該是前者
- [ ] **非技術決策者能讀懂嗎？** → 如果不行，補充上下文或簡化
- [ ] **有沒有空泛的形容詞可以換成具體描述？** → "easy" → "set up in minutes"
- [ ] **是主動語態嗎？** → 被動語態改為主動
- [ ] **有 Problem → Solution → Benefit 結構嗎？** → 至少隱含這個邏輯
- [ ] **AI 相關描述有沒有過度炒作？** → 保持務實
- [ ] **中文文案有沒有翻譯腔？** → 重新用中文思維改寫
- [ ] **技術名詞有遵循保留 / 翻譯規則嗎？** → 對照中英混用規則
